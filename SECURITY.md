# Security Policy

## Supported Versions

This repository is maintained from the default branch. Please test against `main` before reporting a security issue.

## Reporting a Vulnerability

Please do not open a public issue for a suspected vulnerability. Use GitHub's private vulnerability reporting flow from the repository Security tab when available.

Include:

- A concise description of the vulnerability.
- Steps to reproduce or a proof of concept.
- The potential impact.
- Any known mitigations or affected versions.

We will acknowledge reports as soon as possible, investigate, and coordinate a fix before public disclosure.
